How Conversion Tracking Works

A plain-English guide to how Spectra tracks what happens after someone clicks your ad — and how we do it on behalf of your clients automatically.

The Big Picture

What is a "conversion" and why does it matter?

When someone clicks a Google Ad and then does something valuable on your website — fills out a form, buys a product, signs up — that's a conversion. Google Ads uses conversions to understand which clicks are worth paying for.

Without conversion data, Google is flying blind. It shows your ad to everyone equally. With conversion data, it learns who actually buys and shows your ad more to those people. This is what Smart Bidding is — and it's why conversion tracking is the single most important setup task.

The Core Idea You tell Google "this action is worth $X." Google uses that signal to find more people who will do that action. The richer your conversion data, the smarter Google gets.
The Linking Mechanism

How Google connects a click to a conversion

When someone clicks your Google Ad, Google secretly adds a tracking code — called a gclid — to the URL. It looks like this:

https://yoursite.com/pricing?gclid=CjwKCAiA85efBhBbEiwAD7oLQFa...

This gclid is Google's receipt for that specific click. When a conversion fires later, Google matches the conversion back to that original click using the gclid. That's how it knows which ad, which keyword, and which audience led to a sale.

What Spectra does with this Our CaptureClickIds middleware reads the gclid from the URL on first visit and saves it in the session. When the visitor registers, we store it on their user record. This means we can also upload server-side conversions later (e.g. when their first campaign goes live) even if they've long since closed their browser.
Google Tag Manager

GTM is a remote control for your client's website

Normally, adding tracking code to a website means editing the HTML every time. GTM removes that entirely. The client installs one small snippet into their website code once, and then Spectra can add, change, or remove tracking tags remotely — no developer involvement needed.

Think of GTM as an app store installed on their website. We push apps (tags) to it from our end. The client's site automatically runs whatever we publish.

End-to-End Flow

What actually happens when an ad click becomes a conversion

🖱️

1. User clicks the Google Ad

Google appends a gclid to the destination URL. This ID ties the click to everything that happens next.

🌐

2. User lands on the client's website

The GTM snippet (installed once by the client) loads automatically. It fetches the current published container from Google's servers — including all the tags Spectra has configured.

⚙️

3. GTM watches for triggers

GTM continuously watches the page for events we've defined — a form submission, a button click, a thank-you page loading, a minimum time on page, etc.

Trigger listening
🎯

4. Trigger fires — conversion tag activates

When the user does the tracked action (e.g. submits the contact form), the trigger condition is met. GTM fires the Google Ads conversion tag, sending the gclid and a dollar value to Google.

Conversion tag fires
📡

5. Google Ads records the conversion

Google receives the signal: "the person who clicked ad X converted, and this action was worth $Y." It logs this against the original click, keyword, audience, and time of day.

Data recorded in Google Ads
🧠

6. Smart Bidding learns and optimises

Over time (typically after 30–50 conversions) Google's Smart Bidding has enough signal to start predicting who is likely to convert. It automatically bids higher for users who match the profile of past converters — and lower (or not at all) for users unlikely to convert.

Conversion Value Ladder

Why we track multiple events — not just the sale

A sale or sign-up might only happen every few days. Google needs at least 30 conversions per month to optimise well. By tracking multiple points in the funnel — each with an estimated dollar value — we give Google far more data to work with, faster.

Important These values don't have to be exact — they represent the relative importance of each action. A pricing page visit is less valuable than a sign-up, so it gets a lower value. Google uses this to understand the funnel.

Two Tracking Modes

Browser-side vs Server-side conversions

Not all conversions happen while the user is on a webpage. Spectra uses two methods depending on the situation:

🌐 Browser-side (gtag)

JavaScript runs in the user's browser and fires the conversion signal directly to Google Ads in real time.

Used for: Pricing page visit, Sandbox launch, Sign-up. Events that happen while the user is actively on the site.

gtag('event', 'conversion', {
  'send_to': 'AW-XXXX/label',
  'value': 99, 'currency': 'USD'
});

🖥️ Server-side (Conversions API)

Spectra's backend sends the conversion signal directly to Google's API — no browser required.

Used for: Campaign live, 7-day return. Events that happen in our system, not on a webpage. Also more reliable — ad blockers can't block it.

// Backend: RecordSiteConversion job
uploadClickConversions([
  'gclid' => $user->gclid,
  'value' => 80.0
]);
Who Does What

What the client does vs what Spectra does

👤 Client (one-time, 5 minutes)

  • Paste GTM snippet into website <head>
  • Paste GTM noscript snippet into website <body>
  • Save and publish their website

🤖 Spectra (automated)

  • Create Google Ads conversion actions via API
  • Detect which GTM container is on the client's site
  • Provision a new GTM container if none exists
  • Add conversion tags and triggers to GTM
  • Publish the GTM container
  • Verify the tag is live on the site
  • Capture click IDs and upload server-side conversions
  • Refine conversion values over time
What the GTM snippet looks like (the only thing the client installs) The client copies two blocks of code from their Spectra dashboard. Block 1 goes in <head>, Block 2 goes immediately after <body>. Any competent web developer or WordPress admin can do this in under 5 minutes.
Spectra Customer Pipeline

How Spectra sets up tracking for a client's website automatically

1️⃣

Client onboards and connects their Google Ads account

Spectra links their Google Ads customer ID. The nightly maintenance job detects conversion_action_id is empty and queues SetupConversionTracking.

2️⃣

Spectra crawls the client's live website

ConversionSetupService makes an HTTP request to their domain and scans the HTML for an existing GTM container ID (e.g. GTM-XXXXXXX). If found, it uses that. If not, it provisions a new one via the GTM API.

Auto-detects GTM
3️⃣

Creates a Google Ads conversion action via API

A new conversion action is created in the client's Google Ads account. Spectra retrieves the conversion ID and label from the tag snippet.

Conversion action live
4️⃣

Adds the conversion tag + trigger to GTM

Using the GTM API, Spectra adds the Google Ads conversion tag (with the client's conversion ID and label) and a trigger (e.g. "fires on all pages" or "fires on form submit"). The container is then published.

GTM published
5️⃣

Client installs the GTM snippet (if not already present)

If a GTM container was provisioned fresh, Spectra provides the HTML snippet for the client to paste into their site. Once installed, all tags in the container — including the conversion tag — are live immediately.

Summary — The Key Points

  • Google tracks conversions by matching a gclid (click ID) to a later action on the site
  • GTM is installed once on the client's site — after that, Spectra controls all tags remotely via the API
  • Spectra automatically creates conversion actions, adds tags to GTM, and publishes the container — no manual steps
  • Tracking multiple funnel events (not just sales) gives Google more signal and makes Smart Bidding work faster
  • Browser-side events fire via gtag(); server-side events (that happen in Spectra's backend) upload via the Google Ads Conversions API
  • The client's only job is to paste a GTM snippet into their site once — everything else is Spectra
