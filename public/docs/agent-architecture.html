Spectra Media Agent Architecture

How our AI agents think, plan, and act together to build and optimise campaigns across every major ad platform.

1 The Big Picture

What this system actually is, in plain English

Real-world analogy: Think of Spectra as a fully automated advertising agency that lives in a server. When a client brief comes in, a senior strategist reads it and writes a plan (the Strategy Agent). A creative team produces the ads (image and video jobs). A compliance officer checks everything before it goes live (PreLaunchComplianceAgent). Account managers execute the buys on Google, Meta, Microsoft, and LinkedIn (Execution Agents). Then an analytics team monitors performance every day and applies improvements automatically (Optimisation Agents). Nobody sleeps. Nobody forgets.

2 The Orchestrator

The single brain that coordinates everything

Real-world analogy: The orchestrator is the agency operations director. It doesn't write the ads or click the buttons itself — it decides who does what, in what order, and checks that billing is sorted before any work begins. If something goes wrong, it decides whether to retry, escalate, or gracefully fail.

DeploymentService

app/Services/DeploymentService.php

1. Platform resolution
Reads the strategy's platform field and instantiates the correct ExecutionAgent (Google, Facebook, Microsoft, LinkedIn). One line of code picks the right "account manager" for the job.
2. ExecutionContext assembly
Bundles the strategy, campaign, customer, available assets, and budget calculations into an immutable context object — the briefing document handed to every agent.
3. Agent execution
Calls $agent->execute($context) and waits for an ExecutionResult containing platform IDs, warnings, and errors.
4. Result persistence
Saves the full execution plan, all platform IDs (campaign, ad group, asset group), deployment status, and any warnings to the strategy record in the database.

3 Execution Agents

The platform account managers — one per ad network

Real-world analogy: Each execution agent is a specialist account manager who knows one platform inside out. The Google agent knows every campaign type and bidding strategy. The Facebook agent knows Advantage+ audiences and pixel events. They all follow the same workflow template but apply completely different platform knowledge.

Shared workflow — PlatformExecutionAgent (abstract base)

① Validate prerequisites
Checks platform credentials, conversion tracking, minimum budget, and asset availability. If this fails, nothing executes — saves API quota and prevents partial deployments.
② Analyse optimisation opportunities
Checks conversion count, budget level, available assets, and platform features to determine what capabilities are unlocked (Smart Bidding, Advantage+ audiences, Performance Max, etc.).
③ Generate execution plan (AI)
Calls Gemini with a detailed platform-specific prompt. The AI returns a structured JSON execution plan with every step, budget allocation, targeting config, bidding strategy, and reasoning. This is the "thinking" phase.
GoogleAdsExecutionPrompt FacebookAdsExecutionPrompt MicrosoftAdsExecutionPrompt LinkedInAdsExecutionPrompt
④ Review & auto-correct (CampaignReviewAgent)
A second AI pass reviews the plan for common errors. Then PlatformConstraints applies deterministic fixes — e.g. Facebook age_max must always be 65, RSA headlines max 30 chars. No human needed.
⑤ Execute plan
Translates each plan step into real API calls: create campaign → create ad groups → upload assets → link targeting → set bidding. Errors are caught per-step; a recovery plan is AI-generated on failure.

4 Review & Compliance Agents

The quality-control layer that runs before anything goes live

Real-world analogy: Every ad agency has a compliance team that reads every creative before it's submitted to the publisher. Our compliance agents do this automatically — one checks the plan itself for errors (like a proof-reader), and one checks the whole campaign setup (like a legal sign-off). If they flag something, deployment stops.
🔍
Campaign Review Agent
CampaignReviewAgent
Sits between plan generation and execution. Does an AI review of the execution plan for structural errors, then applies deterministic PlatformConstraints fixes (age_max, headline length, objective alignment). Returns a corrected plan.
REVIEW
✅
Pre-Launch Compliance Agent
PreLaunchComplianceAgent
Runs four checks before any deployment begins: (1) ad copy policy — no restricted terms; (2) landing page — HTTP 200 reachable; (3) conversion tracking — tag installed; (4) minimum budget — meets platform floors. One fail = deployment aborted.
COMPLIANCE
🛡️
Platform Constraints
PlatformConstraints
A registry of hard platform rules that are deterministically enforced — no AI involved. Examples: Facebook age_max must be ≥65, Google RSA headlines ≤30 chars, OUTCOME_LEADS must use LEAD_GENERATION optimisation goal. Auto-fixes where possible.
DETERMINISTIC

5 Strategy & Creative Generation

From a client brief to publishable ad creative — fully automated

Real-world analogy: This is the creative department. A senior strategist takes the brief and writes a media plan (strategy generation). The creative team produces copy, images, and videos from that brief. One landscape video and one portrait video are produced per campaign — not per platform — and shared everywhere, the same way a production company shoots one master version of a commercial that then gets cut for different channels.
Strategy Generation
Job: GenerateStrategy → Gemini 3 Pro Preview + Google Search grounding
Ingests campaign brief, customer knowledge base, brand guidelines, and competitor data. Uses extended thinking (temperature 1, topP 0.95) to produce platform-specific strategies: targeting config, bidding direction, creative guidance, video strategy — all in one structured JSON response.
StrategyPrompt Extended thinking Google Search grounding 1 Strategy per platform
Ad Copy Generation
Job: GenerateAdCopy → Gemini · per strategy
Generates headlines and descriptions for each platform, respecting format limits (RSA: 30-char headlines, 90-char descriptions; Facebook: 125-char primary text, 40-char headline). Produces 2–3 variants per strategy for A/B testing.
AdCopyPrompt Platform-aware limits
Image Generation
Job: GenerateImage · 3 images per strategy · $0.07 each
Generates 3 images per strategy using Gemini 3.1 Flash Image (1024px). Images are routed to the correct asset field type at deployment by dimension — landscape images become MARKETING_IMAGE, squares become SQUARE_MARKETING_IMAGE.
ImagePrompt Gemini 3.1 Flash Image Quota-aware dispatch
Video Generation
Job: GenerateVideo · 2 videos per campaign · $2.80 each · Veo 3.1
Generates exactly 2 videos per campaign — not per strategy. Both are shared across all platforms automatically.

Landscape 16:9 — used by Google Ads, YouTube, Performance Max
Portrait 9:16 — used by Meta, Facebook, Instagram

Two-step process: (1) Gemini writes a video script from the strategy brief; (2) Veo 3.1 renders the 8-second video from that script. Only the first signed-off strategy triggers generation — subsequent strategies reuse the same videos.

8 The Bidding Progression Ladder

How campaigns automatically upgrade their buying strategy over time

Real-world analogy: When a new salesperson joins, you don't immediately give them a quota and bonus structure — they need to learn the ropes first. We start campaigns on "get as many conversions as possible" mode. Once they've proven they can convert, we switch to "get conversions at our target cost." Once that's stable, we switch to "maximise revenue return on every dollar spent." The BiddingStrategyProgressionAgent watches the data and promotes the campaign when it's ready.

Requires 50+ conversions / month · Maximises revenue per dollar spent
2

Requires 30+ conversions / month · Hits a target cost per acquisition
1
Maximize Conversions
Starting position · No conversion history required
Automatic progression: The BiddingStrategyProgressionAgent runs weekly. When conversion thresholds are met, it upgrades the campaign via the platform API — no human action needed. If a campaign regresses (conversion count drops), the guard at deployment time will protect against incorrect strategy selection on the next redeploy.

10 Key Design Principles

Why the system is built the way it is

🧱
Determinism before AI
Platform constraints (headline length, age targets, objective alignment) are enforced by code — not trusted to AI. The AI plans, the code enforces. This means no creative hallucination can cause an API rejection.
🔁
Async everything
Every expensive operation — strategy generation, image generation, video rendering, deployment, optimisation — runs as a queued job. The user is never waiting for an API call. The UI shows progress; the work happens in the background.
🛡️
Fail loudly, recover gracefully
Errors are caught per-step, not per-campaign. A failed image upload doesn't abort the whole deployment. Where recovery is possible, an AI-generated RecoveryPlan is attempted. Where it isn't, the error is surfaced with full detail.
📐
Campaign-level resources
Videos are generated once per campaign and shared across all platforms. This prevents the same asset being regenerated 4× for 4 platforms — a 75% cost reduction. The deployment agents look up assets at campaign scope, not strategy scope.
📈
Earned automation
Smart bidding strategies are only applied when the account has enough data to support them. The system starts conservative and earns the right to be aggressive. This mirrors how a skilled media buyer would manage a new account.
🤖
Two-pass AI review
The execution plan is generated by a high-temperature Gemini call (creative), then reviewed by a lower-temperature pass (critical). A second AI checks the first AI's work before any real money is spent. Deterministic constraint fixes are applied last.
